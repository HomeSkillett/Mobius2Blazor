ProgressToggleContentControl

Ready State (Default)

When Ready=True, the content is fully visible and interactive. The spinner is collapsed. Content fades in with a 300ms opacity transition. Source: Mobius_UI_CustomControls_Templates_ProgressToggleContentControl.

Indeterminate (Hidden, keeps layout)

When Ready=False and HideContent=True with ContentVisibility=Hidden, the content becomes invisible but still occupies its layout space. The spinner appears and animates. This prevents layout shift while loading.

Indeterminate — Button (hidden, layout preserved)
Indeterminate — Form field (hidden, layout preserved)
Correction Source

Indeterminate Collapsed (no layout space)

When ContentVisibility=Collapsed, the content is removed from layout entirely. The spinner replaces it. This causes a layout shift but uses less space — appropriate when the loading area doesn't need to maintain its footprint.

Indeterminate Collapsed — Button (content removed from layout)
Indeterminate Collapsed — List content

Indeterminate Disabled (content visible but disabled)

When HideContent=False, the content stays visible but is dimmed to DisabledOpacity (0.35) and made non-interactive. The spinner appears alongside the content. Useful when you want the user to see what's loading without interacting with it.

Indeterminate Disabled — Button (visible, disabled)
Indeterminate Disabled — Form field (visible, disabled)

Spinner Size Variants

The SpinnerSize property (default 20) controls the spinner dimensions. "The standard size works well in the Mode Bar, Tool Bar, vehicle and user lists." Larger sizes for prominent loading indicators.
