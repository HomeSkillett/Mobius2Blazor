Reference Property mapping table

Every WPF Border property has a direct CSS equivalent. Use the token column to find the right variable.

XAML property CSS property Mobius token
BorderBrush border-color var(--InnerLines), var(--BackgroundInvertOutline), etc.
BorderThickness border-width + border-style: solid var(--border-width) (1px), var(--border-width-focused) (2px), var(--border-width-selected) (4px)
CornerRadius border-radius var(--btn-outline-radius) (12px) — or 50% for circles
Background background-color var(--BackgroundPrimary), var(--BackgroundInvertPrimary), etc.
Padding padding var(--padding-all-sm), var(--padding-tb-md), etc.
Margin margin var(--size-6), var(--size-20), or named margin tokens
Opacity opacity var(--opacity-disabled), var(--opacity-readonly), or literal
SnapsToDevicePixels No CSS equivalent needed — browsers handle subpixel rendering.
UseLayoutRounding No CSS equivalent needed.

Watch out Gotchas & ordering rules

BorderThickness order is reversed.
XAML: Left, Top, Right, Bottom
CSS: Top, Right, Bottom, Left (clockwise)
So XAML "0,1,0,0" (left=0, top=1, right=0, bottom=0) → CSS 1px 0 0 0
CornerRadius order matches.
Both XAML and CSS use: TopLeft, TopRight, BottomRight, BottomLeft. No reordering needed.
Margin.All vs Padding.
Mobius XAML uses common:Margin.All="6" on child elements as an attached property. When the child is inside a container, this usually converts to padding on the container in CSS, not margin on the child.
Always use the closest spacing token.
Never hardcode px values when a token is within reach. Check spacingTokens.css for the nearest match.

Pattern 1 Card container

The most common Border pattern. A visible container with background, border, and optional shadow.

Source: Common/Asi.CommonControls.Client.UI/DetailsPopUp.xaml

XAML (source)
<Border Margin="20">
  <Grid MaxWidth="320">
    <common:BoxShadow
        Margin="2"
        Opacity="{...ShadowOpacity}" />
    <Border
        Background="{...BackgroundInvertPrimaryBrush}"
        BorderBrush="{...BackgroundInvertOutlineBrush}"
        BorderThickness="1">
      <TextBlock
          common:Margin.All="6"
          Foreground="{...ForegroundInvertPrimaryBrush}"
          TextWrapping="Wrap" />
    </Border>
  </Grid>
</Border>
HTML + CSS (output)
<!-- XAML: DetailsPopUp.Border -->
<div class="popup">
  <!-- XAML: DetailsPopUp.Border "Border" -->
  <div class="popup__card">
    <p class="popup__message">...</p>
  </div>
</div>

/* --- popup.css --- */
.popup {
  margin: var(--size-20);
}
.popup__card {
  max-width: 320px;
  background-color: var(--BackgroundInvertPrimary);
  border: var(--border-width) solid
          var(--BackgroundInvertOutline);
  box-shadow: 0 2px 8px rgba(0,0,0,0.15);
}
.popup__message {
  padding: var(--size-6);
  color: var(--ForegroundInvertPrimary);
}
What changed: Outer Border (margin-only) became a wrapper div. The separate BoxShadow element collapsed into box-shadow on the card. Inner Border became the card div. SnapsToDevicePixels and UseLayoutRounding dropped — no CSS equivalent needed.

Pattern 2 Circle indicator

Status dots, avatars, activity indicators. Any Border with a very large CornerRadius.

Source: Common/Asi.CommonControls.Client.UI/ActivityStateCircle.xaml

XAML (source)
<Border
    x:Name="_activityStateBorder"
    CornerRadius="100000"
    Background="{...MapBackgroundBrush}">
  <Grid>
    <Border
        Background="{Binding Foreground...}"
        CornerRadius="{TemplateBinding...}"
        Opacity="0.5" />
    <Border
        BorderBrush="{Binding Foreground...}"
        BorderThickness="{TemplateBinding...}"
        CornerRadius="{TemplateBinding...}" />
  </Grid>
</Border>
HTML + CSS (output)
<!-- XAML: ActivityStateCircle._activityStateBorder -->
<div class="activity-circle"></div>

/* --- activity-circle.css --- */
.activity-circle {
  border-radius: 50%;
  background-color: var(--MapBackground);
  border: var(--border-width) solid
          currentColor;
  position: relative;
}

/* Fill overlay at 50% opacity */
.activity-circle::before {
  content: "";
  position: absolute;
  inset: 0;
  border-radius: inherit;
  background: currentColor;
  opacity: 0.5;
}
Key move: CornerRadius="100000" always becomes border-radius: 50%. The two inner Borders (fill + outline) collapse into one element + a ::before pseudo-element. No extra divs needed.

Pattern 3 Divider border (per-side)

A border on only one side, acting as a section separator or divider line.

Source: Common/Asi.Miscellaneous.Client.UI/ModeControls/AboutProduct.xaml

XAML (source)
<Border
    Padding="{...PaddingTopBottomMedium}"
    BorderBrush="{...InnerLinesBrush}"
    BorderThickness="0,1,0,0">
  <Grid>
    <Grid.ColumnDefinitions>
      <ColumnDefinition Width="Auto" />
      <ColumnDefinition Width="*" />
      <ColumnDefinition Width="Auto" />
    </Grid.ColumnDefinitions>
    <!-- content -->
  </Grid>
</Border>
HTML + CSS (output)
<!-- XAML: AboutProduct.Border (top divider) -->
<section class="about-section">
  <!-- content -->
</section>

/* --- about-product.css --- */
.about-section {
  padding: var(--padding-tb-md);
  border-top: var(--line-size) solid
              var(--InnerLines);
}




/* XAML → CSS ordering reminder:
   XAML  "0,1,0,0" = L:0  T:1  R:0  B:0
   CSS   border-top: 1px
   (Reorder from L,T,R,B → T,R,B,L) */
Key move: When only one side has thickness, skip border-width shorthand entirely. Use the specific side: border-top. Cleaner, more readable, and avoids the ordering trap. Use var(--line-size) for dividers, var(--border-width) for container outlines.

Pattern 4 Spacer (invisible border)

A Border with no visible border or background — just spacing. Flatten it out.

Source: Common/Asi.CustomControls.Client.UI/SimpleListItemContentControl.xaml

XAML (source)
<Border
    x:Name="_commandContent"
    Grid.Column="2">
  <Border Margin="-5,0,5,0">
    <ContentControl
        x:Name="_commandContentElement"
        Content="{TemplateBinding...}" />
  </Border>
</Border>
HTML + CSS (output)
<!-- XAML: SimpleListItem._commandContent -->
<!-- Both invisible Borders eliminated -->
<div class="list-item__command">
  <!-- command content -->
</div>

/* --- list-item.css --- */
.list-item__command {
  /* Margin "-5,0,5,0" → XAML L,T,R,B
     CSS: top right bottom left
     = 0 5px 0 -5px */
  margin: 0 var(--padding-all-sm)
         0 -5px;
}

/* Two nested invisible Borders
   collapsed into margin on the
   child element. No wrappers. */
Rule: If a Border has no BorderBrush, BorderThickness, or Background, it's just a spacer. Don't create a div for it. Move its Margin/Padding onto the child element. Every element must have a purpose.

Pattern 5 State-swap overlays

Multiple overlapping Borders toggled by opacity for hover, selection, and toast states.

Source: Common/Asi.CustomControls.Client.UI/NotificationItem.xaml

XAML (source)
<Border
    x:Name="ToastElement"
    Background="{...BackgroundInvertStrongBrush}"
    Visibility="Collapsed" />
<Border
    x:Name="MouseOverElement"
    Background="{...MouseOverPrimaryBrush}"
    Opacity="0.0" />
<Border
    x:Name="SelectionElement"
    Background="{...BackgroundPrimaryBrush}"
    Opacity="0.0" />
HTML + CSS (output)
<!-- XAML: NotificationItem state overlays -->
<!-- Three Borders → one div + CSS states -->
<div class="notification">
  <!-- notification content -->
</div>

/* --- notification.css --- */
.notification {
  background-color: transparent;
  transition: background-color
    var(--transition-mouse-over);
}

.notification:hover {
  background-color: var(--MouseOverPrimary);
}

.notification.is-selected {
  background-color: var(--BackgroundPrimary);
}

.notification.is-toast {
  background-color: var(--BackgroundInvertStrong);
}
Key move: WPF stacks separate elements and toggles opacity. In CSS, use one element with state classes (:hover, .is-selected, .is-toast) that swap background-color. Three overlapping Borders become zero extra divs.

Cheat sheet Quick-reference lookup

When you encounter a Border in XAML, find the matching row and follow the conversion.

When you see… Convert to… Token to use
CornerRadius="100000" border-radius: 50% Literal 50%
CornerRadius="12" border-radius: var(--btn-outline-radius) var(--btn-outline-radius)
BorderThickness="1" border: var(--border-width) solid … var(--border-width)
BorderThickness="0,1,0,0" border-top: var(--line-size) solid … var(--line-size)
Border with no brush/bg No div. Apply margin/padding to child. Nearest --size-* or --padding-* token

common:Margin.All="N" padding on parent or margin on self Nearest --size-* or --padding-* token
