Overlay

Full-screen and sliding overlay components. Unlike Popup (which anchors to a trigger element), Overlay components cover the viewport — either as a semi-transparent modal backdrop or a slide-in side panel. Three patterns from Common Controls and the ConfigurationHistory plugin.

Modal Overlay

Semi-transparent backdrop with centered content. Click backdrop to dismiss. Fade animation uses GeneralMouseOutDuration.

Click the button, then click backdrop to dismiss

Confirm Action

Are you sure you want to proceed with this operation?

XAML: TransparentOverlay — IsActivated toggles Active/Inactive states

Side Panel

Slide-in panel from the right edge. 0.5s quintic easing. Used for configuration forms.

Click the button to slide in panel from the right
Node Configuration

Configure node settings for the selected element.

XAML: NodeConfigurationOverlayControl — IsOverlayOpen toggles OverlayOpen/OverlayClosed states

History Inspector (ConfigurationHistory)

Two-column inspector with timeline on the left and detail entries on the right. 5px divider.

Click to open history. Select timeline items to view details

Blazor Usage

CSS classes for implementing overlay components in Blazor:

.mob-overlay — Base container (required)
.mob-overlay--modal — Centered modal with backdrop
.mob-overlay--panel — Slide-in side panel
.mob-overlay--history — Two-column history inspector
.mob-overlay__backdrop — Click-to-dismiss backdrop
.mob-overlay__content — Modal inner content
.mob-overlay__panel — Side panel container
.mob-overlay__panel-header / __panel-title / __panel-close / __panel-body — Panel sections
.mob-overlay__inspector / __inspector-header / __inspector-body — Inspector layout
.mob-overlay__inspector-timeline / __inspector-divider / __inspector-detail — Timeline and detail areas
.is-open — Show the overlay (add via JS)
