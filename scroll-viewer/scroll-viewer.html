ScrollViewer

Vertical Scroll

Constrained height — vertical scrollbar appears on overflow

This content area demonstrates the Mobius ScrollViewer with vertical scrolling. The scrollbar uses the ScrollBarPrimary color token and transitions to MouseOverPrimary on hover.

In WPF, the vertical ScrollBar track starts at opacity 0 with a translateX(10) offset, then slides in to translateX(4) at full opacity on mouse over. The easing is QuinticEase with EaseOut mode.

The ScrollViewer template uses a 2×2 grid where content spans all cells (row 0–1, col 0–1) and scrollbars overlay on top. The vertical scrollbar sits in column 1, row 0.

Default resting state shows the scrollbar at 0.5 opacity. When the ScrollViewer receives IsMouseOver, both scrollbars transition to 1.0 opacity.

The ScrollBarThumb style sets Background to ScrollBarPrimaryBrush. The thumb template contains a Rectangle (_gripElement) that animates its Fill color between the primary brush and MouseOverPrimary.

This is additional content to ensure the container overflows and the scrollbar becomes visible. The CanContentScroll property is set to True by default, and PanningMode is VerticalFirst for touch devices.

Horizontal Scroll

Constrained width — horizontal scrollbar appears on overflow

This is a single line of wide content that forces horizontal scrolling. The horizontal ScrollBar uses the same thumb styling as the vertical bar — ScrollBarPrimary color with MouseOverPrimary on hover. In WPF, the horizontal track slides in from the bottom edge using translateY(10) → translateY(4) with SmoothEase easing.

Both Scrollbars

Constrained height and width — both scrollbars visible

This demo shows both vertical and horizontal scrollbars active simultaneously. In the XAML template, the ScrollViewer corner cell (Grid row 1, col 1) is empty and transparent — mapped to ::-webkit-scrollbar-corner { background: transparent }.

The ScrollViewer default style sets VerticalScrollBarVisibility and HorizontalScrollBarVisibility to Auto, meaning scrollbars only appear when content exceeds the viewport. This maps directly to CSS overflow: auto.

Both scrollbars share the same IsMouseOver trigger on the parent ScrollViewer, which bumps their opacity from 0.5 to 1.0 simultaneously. In CSS, this is handled by the :hover state on the container affecting the scrollbar pseudo-elements.

The Track component in each ScrollBar template contains DecreaseRepeatButton and IncreaseRepeatButton (page up/down and page left/right commands), but these are styled as transparent buttons — the user drags the thumb or clicks the track to scroll. No arrow buttons are rendered.

Additional overflow content for vertical scrolling. The thumb Border has no explicit CornerRadius in the XAML, but the Rectangle fill creates a rectangular grip. We add a small border-radius in CSS for modern appearance while staying faithful to the intent.

Final paragraph to push content well past the constrained box dimensions and demonstrate the scrollbar intersection corner behavior.

Blazor Usage

Base class: mob-scroll-viewer

Modifiers:

mob-scroll-viewer--vertical — vertical only (overflow-x hidden)

mob-scroll-viewer--horizontal — horizontal only (overflow-y hidden)

mob-scroll-viewer--constrained — applies max-height via --scroll-height custom property

mob-scroll-viewer--fill — fills parent height (100%)
