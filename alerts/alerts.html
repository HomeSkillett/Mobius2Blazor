Alerts & Notifications

Mobius has five distinct patterns for communicating alerts, status changes, and confirmations to the operator. Each serves a different purpose and has its own XAML control.

Patterns at a Glance

Toast

NotificationItem.xaml — IsToast trigger
Warning
GPS Signal Lost
Vehicle 12 has lost GPS connectivity in Field B.
2 min ago

Transient slide-in from the right. Auto-dismisses or can be manually closed. Used for real-time event notifications (vehicle faults, GPS loss, etc.).

View Toast →

Notification Item

NotificationItem.xaml — default mode
New Critical ERR-4081
E-Stop Activated
11:42 AM

Persistent list item inside the Notification Panel. Expandable message, action buttons, dismiss, and color-coded level bar on the left edge.

View Notification Item →

Notification Panel

InternalNotificationPanel.xaml

Full sidebar panel with filter tabs, vehicle filter, time filter, dismiss-all, and a history section with sortable grid.

View Notification Panel →

Active Alert

ActiveAlertUI.xaml — Agriculture plugin
 Section coverage complete — moving to next row

Full-width banner with a progress bar counting down the timeout. Four visual states: Info, Warning, Error, and Inactive/OK.

View Active Alert →

Confirmation Dialog

OkCancelDialog.xaml
Confirm Action
Are you sure you want to end the current task?

Modal dialog for binary confirmations. Contains a title, message, and OK / Cancel buttons. Blocks interaction until dismissed.

Level Enums & Color Mapping

Two enums drive the color system. MessageDeliveryLevel is used by Toasts and Active Alerts. NotificationLevel is used by the Notification Panel and NotificationItem.

MessageDeliveryLevel NotificationLevel Icon Color Background
