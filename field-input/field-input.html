FieldInput

The primary text input component in Mobius (382 uses across the codebase). Combines a label, text input, optional unit display, and validation feedback into a single control. Supports numeric entry with min/max ranges, unit-of-measure conversion, hex input, and keyboard commit/revert behavior. Extends Control and wraps TouchTextBox. Source: FieldInput.xaml + FieldInput.cs.

Horizontal Template (Default)

The most common layout. Four-column grid: label, status icon, text input, unit label. Label column uses SharedSizeGroup so labels in the same form section align to the widest. Default label min-width is 130px. XAML key: Mobius_UI_Common_Templates_FieldInputHorizontal.

Interaction States

Visual States

FieldInput has three validation visual states controlled by the ShowStates VisualStateGroup. Warning shows an orange border + exclamation icon when input is out of range. Working shows a green border + checkmark when the value has been edited but not yet committed (Enter to commit, Esc to revert). Error/invalid is from the base TextBox validation template.

Unit Label

Displays the current unit of measure (ft, mm, deg, sec, etc.) to the right of the input. Driven by the PhysicalType property and the app's UnitsManager (metric vs imperial). Visibility controlled by ShowUnitLabelStates. Dims to 50% opacity when the field is disabled.

Helper Text

Descriptive hint text below the input, aligned to the input's left edge (not the label). Used in configuration forms to clarify expected format. Not part of the FieldInput control itself, but a common companion pattern in the XAML views (e.g. GeneralConfiguration.xaml).

Auto-Width Label

When LabelMinWidth="0", the label column auto-sizes to its content instead of the default 130px minimum. Wrap sibling fields in .mob-field-group to use CSS Grid subgrid alignment, mirroring XAML's SharedSizeGroup behavior. Used in compact panels like Position History settings.

Vertical Template

Label stacked above the input. Label row is fixed at TouchSize (44px) height. Warning/working icons slide in to the left of the label text via an ExpanderItem animation. Unit label sits to the right of the input in the bottom row. Used in forms where horizontal space is tight or fields need more visual separation. XAML key: Mobius_UI_Common_Templates_FieldInputVertical.

ft

Inline Template

Compact single-line layout. Label sits flush left, input fills remaining space, and unit label + status icon overlay inside the input's right edge. Input right-padding is dynamically sized to prevent text from running under the overlaid elements. Uses OnDemandTextBox style (borderless until interaction). XAML key: Mobius_UI_Common_Templates_InlineFieldInput.

Style Variant: Invert

Swaps input background and border to the "invert" color tokens for use on panels with BackgroundInvert backgrounds. Style Variant: On-Demand

Input appears borderless and transparent until hovered or focused, then reveals the standard input chrome. Smaller icon column (20px vs 24px) and tighter padding (3px 0). Good for data-dense views where full input borders would be visually noisy. XAML key: Mobius_UI_Resources_Styles_OnDemandFieldInput.

Style Variant: Large Title

Vertical layout with a heading-sized on-demand input (36px / Roboto Light). Combines OnDemandTextBox + TextBlock_HeadingLg styles. Unit label is always hidden. Negative input margins pull the text closer to the label for a tighter title-editing feel. Used for editable entity names (maps, projects, configurations). XAML key: Mobius_UI_Resources_Styles_FieldInputLargeTitle.
